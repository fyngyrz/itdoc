11.4.68 - Script

The script operator provides for running sequences of operations that modify image pixels on the selected area. Provision of a number of non-image commands offers an additional means to manipulate values, create text strings, loop, use conditionals, and create functions.

11.4.68.1 - Creating Basic Scripts

You can edit scripts in the operator dialog, however, if the available space seems too cramped, use the ⇒ Edit button to open a larger editing dialog.

  • You can build a script of image commands one-per-line by using the →Script button in various operators
  • You can enter script commands directly in the text box in the Script operator
  • You can enter scripts in a larger edit window accessed from the Script operator
  • You can load previously saved scripts from mass storage using Load/Save

You can save scripts you have created or modified. Scripts consist of one or more commands that are executed from top to bottom, one operation per block. A block can consist of a single line containing one command, or one of multiple blocks in a line separated by the : (colon) character.

Each image command in the script is checked for correctness. If any of the parameters for the image command are out of range, the operation won't execute. If the image command is not recognized, it will be ignored and the results of any other operations in the script will be undone.

When typing commands directly into the script box in the first block on a line, iToolBox will make suggestions in the prompt area below the script box as to what the possible command might be based on the characters you've typed so far. When you have typed enough characters such that there is only one possible candidate, that candidate will be the only one shown, and typing a tab will auto-complete the keyword for you. At this point the prompt area changes to a guide to the parameters for that command. Here's an example of how that can go starting with a blank line...

You type Preview shows Line contains
b
bend,brushstroke,burn
  b
u
burn
  bu
tab
burn [Amount=0...100]
  burn

...at this point, the prompt area shows you have one value to provide to burn, so you type a space and that value, and you're done.

Errors will result in both a report of the error lines and a listing of the correct parameters and ranges for those operations at the top of the script when the script is executed. Once the errors are cleared, the error report will disappear upon script execution and the script's result will be accepted.

You may place one command on a line (this is a good idea with image commands so you can take advantage of the suggestions feature), or you can place more than one command per line by separating them with a ":" (a colon.) Each command is a block; so one command on a line represents one block, and several commands separated by colons represent multiple blocks.

You can disable any block in a script by turning them into a comment by simply placing a # character at the beginning of the block(s.) Comment blocks are ignored. Keep in mind that the start-of-block sequences "#!" and "#*" are reserved and must not be used in comments. Examples:

#a=1One line of one block, commented out
a=1:#b=2:c=3One line with three blocks, b=2 commented out

This operator provides syntax highlighting. You can set the colors via the Script Settings dialog, available from the Settings button.

11.4.68.2 - Operator Commands

Operator commands generally consist of a keyword, followed by any required parameters, all separated by spaces, contained in one block. In the list below, the green information is what you type into the script; the red information is just to let you know what the parameters mean. In addition, the ellipsis (...) indicates a range, and is not entered into the script itself. Here's the information for the alphamod operator command:

alphamod [Amount=0...255] [Mode=0...9]

So to use the alphamod command to create an alpha level of 128, you would follow that guidance and enter the following in a block:

 alphamod 128 0

For checkboxes, a value of 0 means the box is unchecked; a value of 1 means is is checked.

For drop-down selectors, the first entry is 0, and each entry thereafter is a value increased by one. Or another way to look at it is, count the entries until you reach the one you want, then subtract one.

11.4.68.3 - Variables

Numeric variables are created by writing a variable name, such as x or myvariable, followed immediately (without an intervening space) by an equals sign, and then the contents you want to put in the variable.

Text variables are created by writing a variable name, such as x or MyVariable, followed immediately (without an intervening space) by a tilde (~),and then the contents you want to put in the variable.

Variable Rules:

  • Variable names cannot begin with numbers: 2x is an illegal variable name, x2 is legal.
  • Variable names cannot contain an '=' character
  • Variable names cannot contain a '~' character
  • Variable names are all lower case, even if you write them otherwise

There are two types of information you can put in a variable. Real numbers (decimal values, optionally with fractional parts) numbers, and text. Examples:

v=5A variable named v that contains the whole number 5.
Will always produce the value 5 in any numerical context, and the character "5" in any character context.
Whole number use example:
value=128
alphamod value 0
...evaluates to:
alphamod 128 0
yvalue=2.7A variable named yvalue that contains the real number 2.7.
Will produce the value 2.7 in a real number context; will produce the value 3 in an integer context (that's 2.7 rounded to the nearest whole number), and the character string "2.7" in any character context.
Real number use example:
MyScale=1.1
text Courier 100 100 MyScale testing
...evaluates to:
text Courier 100 100 1.1 testing
mykey~Testing 1, 2, 3A variable named mykey that contains the characters Testing 1, 2, 3
Text variables may only be used where an operator expects a character string, or when being concatinated into other text variables. If a text variable is used in a numeric context of any kind, this will result in an error. Optional double quotes surrounding the string cause the variable to be loaded without evaluation; without quotes, the + (plus sign) character indicates concatenation. Strings may contain literal double quotes.
Character string use example:
myrandomkey~"My cat Twinkie"
random 0 0 0 100 myrandomkey
...evaluates to:
random 0 0 0 100 My cat Twinkie

Because image command blocks are specified on a per-token basis by separating each token with a space, when a character value is required and it is not at the end of the set of tokens, the only way you can specify a token containing spaces is to use a variable as an intermediary.

For instance, the text area selection command requires a font name. A single-word font name like Courier is fine, as it contains no spaces, so you can just write the command literally, whereas multiple word font names will require the use of a variable. Examples:

text Courier 100 100 1.1 testingA literal, single-word font name
font~Marker Felt
text font 100 100 1.1 testing		A multi-word font name contained in a variable

The following demonstrates the effect of surrounding a string with double quotes:

x~Marker+ +FeltResult: Marker FeltMarker, a space, and Felt are concatenated.
x~"Marker+ +Felt"Result: Marker+ +FeltString literal. No concatenation occurs.

11.4.68.4 - Text Expressions

Text expressions may be used wherever a variable is being set.

Text expressions may contain variables and/or literal values. Variables have priority over literal values; for instance, if there is no variable named x, and you write var=x+foo, var will contain xfoo. However, if the variable x contains something, for example a 2, then var will contain 2foo.

Text expressions may contain the operator + for concatenation. To express a literal + in a string where you still need to use concatenation, use a digraph — two plus signs in a row: var=two++two which evaluates to var=two+two, whereas var=two+two evaluates to var=twotwo. Alternatively, if you surround the variable assignment value with quotes, no evaluation will take place: var~"two+two" evaluates to two+two, but in this case you cannot perform concatenation as the string is treated as a literal.

In a text expression, you can refer to a numeric variable with its first letter capitalized, and the result will be the expression that created the numeric result, rather than the result itself:

test=2+3
tt~test		result: tt is "5"Numeric value of variable test as a string
test=2+3
tt~Test		result: tt is "2+3"The expression that created variable test as a string
test=2+3
tt~Test+=+test		result: tt is "2+3=5"Both, concatenated as a string

Neither text expressions or string literals may contain single colons, as these indicate the end of one block and the beginning of another. To represent a colon in a text expression or literal, use the digraph :: instead of a single colon character.

Text expressions may contain variables as concatenated elements, as long as there is no additional content between any + concatenation operator(s) and the variable name. Here's how that works:

hoo=5
x~boo+hoo+test		Result: boo5test"hoo" evaluated as variable.
hoo=5
x~boo+hoo +test		Result: boohoo test"hoo " evaluated as string.
hoo=5
x~boo+ hoo+test		Result: boo hootest" hoo" evaluated as string.
hoo=5
x~boo+ hoo +test		Result: boo hoo test" hoo " evaluated as string.
hoo=5
x~"boo+hoo+test"		Result: boo+hoo+teststring literal; no evaluation.

Double quotes only have special meaning when they surround an entire text expression. Otherwise, they may be freely used within string expressions and will be evaluated as literal double quote characters:

baggy~droopy
x~baggy "pants"		Result: baggy "pants"Unquoted, evaluated as string.
baggy~droopy
x~"baggy "pants""		Result: baggy "pants"Quoted, unevaluated.
baggy~droopy
x~baggy+ "pants"		Result: droopy "pants"Unquoted, evaluated as string.
baggy~droopy
x~"baggy"+ pants		Result: "baggy" pantsUnquoted, evaluated as string.
baggy~droopy

11.4.68.5 - Numeric Expressions

Numeric expressions may contain the arithmetic operators + for addition, - for subtraction and negation, * for multiplication, and / for division.

They may also contain the comparison operators ==, !=, <, >, <=, >=

Numeric expressions may not contain any spaces when used in the context of a parameter in an operator command; this allows expressions to be used as numeric parameters to an operator command where the operator command's individual parameters are separated by spaces.

Example numeric expressions:

a=5
y=a+2		The variable y now contains 7
a=5
y=a+-2		The variable y now contains 3
b=7
foo=12
alphamod foo+b*2-1 0		 The result here is this command: alphamod 37 0
Here's how that happened: First, b is added to foo, which results in 19. Then 19 is multiplied by 2, which results in 38. Finally, 38 has 1 subtracted from it, which results in 37

Expressions are evaluated in a strict left-to-right order. Multiplication and division do not have priority over addition and subtraction. Operations occur in the order they are written.

If you need to control operation order, use ( and ) parentheses. For example:

x=2+2*3The variable x now contains 12
x=2+(2*3)The variable x now contains 8
11.4.68.5.1 - Unary Operations

An unary operation is one that directly affects the following value. There is only one unary operation:

-valueNegation

11.4.68.9 - Custom Functions

You can create custom functions. Custom functions are quoted text variables containing one or more command blocks which are separated by ; (semicolon) characters (not colons!) You can put anything in a custom function that you can in any other block.

Here's a custom function that will add one to the variable a, and an example of its use:

inca~"a=a+1"
void=func(inca)

Here's one that will colorfill in orange:

orangefill~"r=255;g=128;b=64;colorfill r g b;#orange!"
void=func(orangefill)

Custom functions are evaluated when they are invoked. This makes them dynamic instead of static. Examples:

myfunc~"a=b+c":# function definition
b=1:c=2:# initial setup
void=func(myfunk):# variable a will now be 3
b=2:void=func(myfunk):# variable a will now be 4
Tip: If you need to represent an actual semicolon in a custom function, use a digraph: ;;

11.4.68.12 - Looping

Loops are built using two commands; a starting command, loop([expression]), that can include an optional expression specifying a maximum looping count, and an ending command, endloop([expression]), that can include an optional expression defining a terminating condition. Neither, either or both my be present.; A loop without any expression specified in either the starting or ending commands will run once.

The loop(count), if provided, takes priority over any specified ending condition; the loop will terminate before the ending condition is satisfied if it executes the maximum number of loops initially specified.

If you may need a loop to run zero times, surround it with an if() expression, like this:

if(n>0):loop(n):a=a+1:#etc.:endloop():endif

Be careful not to set up a condition where the loop cannot end naturally, such as:

loop():a+a+1:endloop(1)

In that case, the loop will not terminate until the number of blocks executed reaches the maximum count set by the pre-defined variable m_maxops

Caution: You can change the value of the m_maxops variable in your script, or in your script presets. If you do so unwisely, you can set up a condition where a script will never end, practically speaking.
