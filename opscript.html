11.4.65 - Script

Provides for running sequences of operations that modify image pixels on the selected area.

You can enter a script directly in the box, or you can load one from the filesystem. You can also save scripts you have created or modified. Scripts consist of a sequence of command (operation) lines that are executed from top to bottom, one operation per line.

Each line in the script is checked for correctness. If any of the parameters for the operation are out of range, the operation won't execute. If the operation is not recognized, it will be ignored and the results of any other operations in the script will be undone.

Errors will result in both a report of the error lines and a listing of the correct parameters and ranges for those operations at the top of the script. Once the errors are cleared, the error report will disappear upon script execution and the script's result will be accepted.

If you'd like to see the parameters for a particular operation, just enter the operation name followed by a question mark, for instance edgedetect ?

You can disable any line or lines in a script by turning them into a comment by simply placing a '#' character at the beginning of the line(s.) Comment lines are ignored. Keep in mind that the start-of-line sequences "#!" and "#:" are reserved and must not be used in comments.

This operator uses syntax highlighting. You can set the colors via the Meta Defaults dialog.

11.4.65.1 - Available Script Commands

Script commands generally consist of a keyword, followed by any required parameters, all separated by spaces. In the list below, the green information is what you type into the script; the red information is just to let you know what the parameters mean. In addition, the ellipsis (...) indicates a range, and is not entered into the script itself.

So for instance, to use the alphamod operation to create an alpha level of 128, you would enter only this on a line:
 alphamod 128

Some operators do not have supporting scripting commands. They are indicated by red, struck-out text:
 operator 

For checkboxes, a value of 0 means the box is unchecked; a value of 1 means is is checked.

For drop-down selectors, the first entry is 0, and each entry thereafter is a value increased by one. Or another way to look at it is, count the entries until you reach the one you want, then subtract one.
